# Contact Telegram Bot

Telegram-бот для игры "Контакт".

Бот ведет механику раунда:
- хранит секретное слово ведущего в личке;
- показывает меню одной основной командой `/contact`;
- собирает лобби;
- помогает выбрать ведущего;
- принимает вопросы через личку;
- запускает классический "Контакт" с отсчетом 5 секунд;
- открывает следующую букву, если контакт сошелся;
- принимает попытку назвать все слово;
- умеет завершать игру вручную.

## Основные команды

- `/contact` - главное меню игры в группе и справка в личке.
- `/say текст` - админская команда: бот пишет текст от своего лица в группе.

## Как играть

1. В группе кто-то пишет `/contact` и нажимает `Создать игру`.
2. Игроки нажимают `Войти`.
3. Админ выбирает ведущего кнопкой `Выбрать ведущего`.
4. Ведущий нажимает `Слово в личку` и отправляет боту секретное слово.
5. Админ нажимает `Старт`.
6. Участник нажимает `Задать вопрос`, а потом в личке отправляет:

```text
слово :: определение
```

Пример:

```text
солнце :: Это не такой желтый шарик?
```

7. Другой участник нажимает `Контакт` и отправляет боту в личку слово, которое он понял.
8. Если ведущий не успел отбить вопрос и слово контакта совпало со словом автора вопроса, бот открывает следующую букву.
9. Когда кто-то знает все слово, он нажимает `Назвать слово` и пишет его боту в личку.

## Поведение бота

- Тон сообщений короткий, живой и понятный.
- Ведущий не видит секретные слова вопросов в группе.
- Бот не проверяет грамматически, что слово точно является нарицательным существительным в именительном падеже: это остается на честность ведущего.
- Режим контакта классический: бот ждет 5 секунд.

## Локальный запуск

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
$env:BOT_TOKEN="your_botfather_token"
python main.py
```

## Deploy на Render

Проект уже содержит `render.yaml`.

1. Запушь код в GitHub.
2. Создай сервис в Render из репозитория.
3. Добавь переменную окружения `BOT_TOKEN`.
4. Запусти деплой.

Health endpoint: `/health`
